# Incremental Build Graph Convergence Failure

Fix the Go implementation under `/app/environment` to correctly reconcile graph edges.

Prune stale dependency edges cleanly in the database representation.

Compute correct fingerprints of build actions using Go interface signatures.

Synchronize the registry while cleaning up stale physical output files.

Static or manual output writes are insufficient; the build engine's source code must be corrected.

The build engine pipeline must correctly regenerate the artifacts and manifest dynamically on subsequent build cycles.

Library-level and module-level behaviors must be correct, not just the CLI output.

Pruned/removed targets must not exist on disk (existence is false).

Specifically, graph reconciliation must correctly result in a count of exactly 2 active edges when the edge configuration is updated.

The build database is a sqlite3 database.

The engine must produce the compiled build engine binary.
