We ship a headless rollback-style netcode lab under `/app`. Straight-through seeded runs agree run-to-run, but after a simulated disconnect/reconnect, a mid-replay resume, or a replay blob export then import, the lab’s fingerprints disagree with a clean uninterrupted run for the same scripted commands. Your job is to make the interrupted paths match the uninterrupted baselines again without weakening determinism elsewhere.

The scripted timelines live in `/app/data/seed_scenarios.json`. Run the matrix driver so it writes `/app/output/matrix_report.json` and `/app/output/trace_bundle.jsonl`.

The matrix report uses integer `schema_version` **4** for the current contract, integer `reader_lowest_revision` (smallest replay blob revision the reader still accepts), top-level `rng_channel_totals` with separate integer counters for `world` and `net`, integer `replay_format_revision` (newest writer revision the reader still accepts while loading older blobs), and a `scenarios` object keyed by scenario id. Use a **per_scenario** layout: one nested object per pack entry. Every scenario object must include string `state_digest_hex` (64 lowercase hex characters, 256-bit digest of the canonical projection) and non-empty string `tail_projection` (deterministic summary of the last few canonical ticks). Depending on what each pack row exercises, also include the matching keys: rows that run the driver twice for repeatability must include boolean `duplicate_match` plus string digests `straight_run_a` and `straight_run_b`; rows that record then replay through the blob pipeline must include boolean `roundtrip_match` plus string `roundtrip_digest` alongside the uninterrupted reference digest in `state_digest_hex`; rows that splice an uninterrupted prefix with a decoded suffix must include boolean `resume_ok`, string `baseline_digest_hex`, and string `state_digest_hex` for the stitched path; rows that load an older blob format must include integer `legacy_revision_read`. Each scenario object may include string `mode` echoing the pack’s mode label for debugging.

Automated verification may compile the driver under constrained build profiles so that a fix touching only one subsystem still leaves at least one scenario in the pack failing until the full integration work is done. Closed grading compares digests and `rng_channel_totals` to verifier-held expectations for this `schema_version` and pack revision; those literal expectations are not reproduced in this prompt, but your outputs must match them when the implementation is correct.

Each line in `/app/output/trace_bundle.jsonl` is one JSON object with string `scenario_id`, integer `tick`, two-element integer array `merged_dx` (player0 then player1 after reconciliation), two-element integer array `positions`, and integer `entropy_mix` from the world channel’s folded contribution. Keep world and net RNG draws separated in the report counters. Older recorded blobs at revision `1` must still load when the writer moves forward, as reflected in `replay_format_revision`. After any source or build-system changes, rebuild the `netplay_matrix` binary and install it at `/app/bin/netplay_matrix`; grading runs that binary itself (passing `--out /app/output --pack /app/data/seed_scenarios.json`) and reads the outputs from `/app/output/`.

The build exposes two CMake cache variables, `TB_TIMELINE_MODE` and `TB_REEXEC_MODE`, each selecting between two compile-time variants of independent subsystems in `src/`. Grading compiles the driver under several combinations of these values, so both branches of every `#if TB_*_MODE` ladder in `src/` must remain compilable — collapsing a ladder or hard-coding one branch is not a valid fix.
